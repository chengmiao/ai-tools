# 经验教训目录

按 feature 建立经验链，记录决策演变历史。

## 目录结构

```
exp/
├── README.md              ← 本文件
├── <feature-name>/        ← 每个 feature 一个子目录
│   ├── v1.md              ← 初始经验
│   ├── v2.md              ← 后续迭代（引用 v1）
│   └── ...
```

## 经验文件模板

```markdown
# <feature-name> v<N>: <简短标题>

**Parent:** v<N-1>（首版写 "无"）
**Feature:** <对应的 feature 文件路径>
**日期:** <YYYY-MM-DD>
**类型:** initial | iteration | fix | optimization

## 本次做了什么决策、为什么

<决策内容>

## 踩过的坑和解决方案

<问题和方案>

## 与上一版经验的关系

<说明本次经验如何延续/修正/推翻了前一版的结论>
（首版写：这是该 feature 的首次经验记录）

## 下次遇到类似情况的建议

<建议>
```

## 规则

- **一个 feature 一个子目录**: 目录名与 feature 名保持一致
- **必须引用 parent**: 非首版必须标注前一版本号并说明演变
- **只追加不修改**: 已有文件不可修改，新经验作为新版本追加
- **跨 feature 引用**: 用 `参见 exp/<other-feature>/v<N>.md` 格式
- **不得跳过**: "没什么特别的"不是理由，每次必须至少三句话
