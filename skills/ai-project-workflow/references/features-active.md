# Feature 文件格式

## 文件命名

`<priority>-<short-name>.md`，如 `p0-base-ingestion.md`

## 模板

```markdown
# Feature: <名称>

## 背景

<为什么需要这个功能>

## 目标

<具体要实现什么>

## 验收标准

- [ ] <可验证的条件 1>
- [ ] <可验证的条件 2>
- [ ] <可验证的条件 3>

## 依赖

- 前置 feature: <无 / feature 文件名>
- 外部依赖: <无 / 具体依赖>

## 优先级

P0 / P1 / P2 / P3

## 预估复杂度

S / M / L / XL

## 备注

<其他需要注意的事项>
```

## 管理规则

- `active/` 中保持 2-5 个 feature
- 完成后移入 `done/`（保留文件，不删除）
- 验收标准必须是可客观验证的（能运行命令 / 能看到输出）
