# 实现方案目录

## 文件命名

`<feature-name>.md`，与 feature 文件名对应

## 模板

```markdown
# Plan: <Feature 名称>

**Feature**: <对应 feature 文件路径>
**日期**: <YYYY-MM-DD>
**状态**: draft / approved / in-progress / done

## 前置条件

- <context 约束引用>
- <依赖确认>

## 实现步骤

### Step 1: <标题>
- **文件**: <确切文件路径>
- **操作**: <具体代码变更>
- **验证**: `<运行命令>` → 预期输出: `<输出>`
- **回退**: <失败时怎么回退>

### Step 2: <标题>
...

## 风险点

- <可能的问题及应对>
```

## 规则

- 粒度: 每步 2-5 分钟，禁止占位符
- 必须包含: 文件路径 + 代码 + 验证命令 + 回退方案
- 用户明确确认后才能开始 Work
